The PyPI wrapper package (`pgdesign/pypi/`) has `VERSION = "0.1.0"` hardcoded but pyproject.toml is at a higher version. The wrapper constructs a GitHub Release download URL using this VERSION, which 404s because no v0.1.0 release exists. The VERSION constant must be kept in sync with pyproject.toml, or better yet, read dynamically from package metadata via `importlib.metadata.version()`.
